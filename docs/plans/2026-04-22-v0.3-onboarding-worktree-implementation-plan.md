# v0.3 Proactive Agent And Worktree Enablement Implementation Plan

## Goal

Make DoneGate easier to start correctly in multi-worktree agent workflows by improving hook installation for git worktrees and generating repo-local onboarding guidance for Codex and Hermes.

## Scope

- make `bootstrap` resolve the correct hooks directory in normal repos and linked git worktrees
- generate repo-local onboarding assets during bootstrap
- add an `onboarding` CLI command that reports current branch, active-task context, and agent setup hints
- provide Codex and Hermes focused guidance without adding a second configuration system

## Deliverables

- `.donegate-mcp/env.sh` with repo-local environment exports
- `.donegate-mcp/onboarding/codex.md`
- `.donegate-mcp/onboarding/hermes-mcp.yaml`
- `donegate-mcp onboarding --repo-root . --agent codex|hermes`

## Worktree Behavior

- resolve the hook target via `git rev-parse --git-path hooks`
- keep managed hook installation behavior unchanged for normal repos
- support linked worktrees without relying on a physical `.git/hooks` directory under the worktree root

## Onboarding Behavior

- include current branch and resolved active task for the repo root
- include the next recommended command when no branch task is active
- include repo-local file paths so agents can bootstrap themselves with less manual lookup

## Test Plan

- service test for bootstrap in a linked git worktree
- service test for onboarding payload with branch-scoped active task
- CLI test for `onboarding`
- CLI/bootstrap test proving onboarding files are generated
