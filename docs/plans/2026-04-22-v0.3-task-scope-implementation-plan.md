# v0.3 Task Scope Ownership Implementation Plan

## Goal

Teach DoneGate to reason about whether the current active task actually owns the changed files in the repository, instead of assuming any active task is sufficient coverage.

## Scope

- add optional task-owned path scopes to the task model
- accept scopes through CLI and MCP task creation/protocol updates
- normalize task scopes against a stable repo root
- have supervision report covered and uncovered files for the active task
- return `task_mismatch` when the active task exists but does not cover all changed files

## Design Notes

- Task scopes will be stored as repository-relative paths or glob expressions.
- Plain path entries behave like owned prefixes.
- Glob entries behave like include patterns over git-relative changed paths.
- Supervision should stay lightweight and avoid policy enforcement in this slice.

## Test Plan

- service test proving uncovered files produce `task_mismatch`
- service test proving scope inputs normalize to stable repo-relative values
- CLI test proving `--owned-path` flows through task creation and supervision
- full pytest regression after the focused slice passes
