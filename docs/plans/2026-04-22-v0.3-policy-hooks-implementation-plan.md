# v0.3 Richer Supervision States And Policy Hooks Implementation Plan

## Goal

Upgrade DoneGate supervision from coarse repository tracking into a more actionable read model that can drive hook policy decisions without introducing a background daemon.

## Scope

- extend supervision beyond `clean`, `needs_task`, `task_mismatch`, and `tracked`
- surface policy-oriented states for stale verification, stale docs, and spec revalidation
- attach stage-aware policy guidance for `pre-commit` and `pre-push`
- make managed hooks consult supervision before running self-tests

## Proposed States

- `clean`: no relevant repo issue is currently visible
- `needs_task`: changed files exist but no active task is bound
- `task_mismatch`: changed files fall outside the active task scope
- `needs_revalidation`: the active task has spec drift and must be revalidated
- `stale_verification`: the active task covers the current diff but still lacks passing verification for the working state
- `stale_docs`: the active task has passing verification but doc sync is still outdated
- `tracked`: changed files are covered and no stronger supervision warning applies

## Policy Shape

- `pre-commit`
  - block on `needs_task`, `task_mismatch`, `needs_revalidation`
  - warn on `stale_verification`, `stale_docs`
  - allow on `clean`, `tracked`
- `pre-push`
  - block on `needs_task`, `task_mismatch`, `needs_revalidation`, `stale_verification`, `stale_docs`
  - allow on `clean`, `tracked`

## Test Plan

- service test for `needs_revalidation`
- service test for `stale_verification`
- service test for `stale_docs`
- hook test showing `pre-commit` blocks on scope mismatch
- hook test showing `pre-push` blocks on stale docs
